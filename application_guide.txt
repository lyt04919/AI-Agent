【小米百万亿token活动 - 申请填写建议】

项目解决的核心痛点：
1. 长尾复杂问题处理效率低：传统机器人只能处理简单问答，面对跨系统查询（如查订单、算退费）的复杂诉求时需人工介入。
2. 知识孤岛与信息碎片化：企业政策文档长且散，人工检索成本高。
3. 跨部门协同成本高：缺乏自动化的意图拆解和任务分发机制。

核心逻辑流：
采用“路由-检索-执行-审查”的多 Agent 协作架构：
- Agent 1 (Router): 进行长链推理，拆解用户复杂意图。
- Agent 2 (RAG): 深入检索内部政策库，支持长文本理解。
- Agent 3 (API Executor): 自动调用外部 API 获取订单实时数据。
- Agent 4 (QA Checker): 通过多智能体辩论（Debate）机制，核对政策与数据，生成最终合规回复。
